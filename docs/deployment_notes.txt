# Guynode — Deployment Notes

## Hosting
- Platform: Cloudflare Pages
- Build command: npm run build
- Output directory: dist
- Node version: 22+

## Routing
- SPA routing via query parameters (?view=CATALOG etc.)
- SPA fallback: public/_redirects (/* /index.html 200)
- Legacy URL redirects: public/_redirects (301s for
  all old .html pages — see file for full mapping)

## Build pipeline
- Prebuild: scripts/validate-content.ts (Zod schema
  validation on datasets.json and blog/index.json)
- Prebuild: scripts/generate-sitemap.js (regenerates
  public/sitemap.xml and public/robots.txt)
- Build: vite build → dist/

## Environment variables
- NODE_ENV: set to 'production' automatically by
  Cloudflare Pages on build
- No API keys required — application has no runtime
  backend or AI dependencies

## GCS asset pipeline
- Bucket: guynode-public-assets (public, wildcard CORS)
- Conversion: scripts/convert-and-upload.py
  (ogr2ogr shapefile → GeoJSON + gsutil upload)
- Run with: python scripts/convert-and-upload.py
  --assets-dir /path/to/assets --update-json

## Known limitations
- MapViewer chunk: ~1.9MB (GeoRaster — pre-existing,
  tracked for future code-splitting)
- GeoJSON assets must be uploaded via
  convert-and-upload.py before map viewer layers
  display data
